# Spec: recodex v0.2.0 ContextSeek-Inspired Refactor

## Objective

Refactor `recodex` from a Codex-first retrospective CLI into a small, extensible
AI coding session ingestion and improvement system.

The goal is not to copy ContextSeek. ContextSeek is a broader context memory
platform with a dashboard, storage backends, DataPlugs, watcher, and evolution
pipeline. `recodex` should stay narrower:

```text
recodex = AI coding session importers + retrospective analysis + improvement exports
```

The v0.2.0 refactor should make these capabilities reusable:

- import AI coding sessions from multiple tools
- incrementally scan files without duplicate writes
- generate evidence-backed retrospectives and improvement candidates
- export accepted improvements as local workflow artifacts, especially `SKILL.md`
- optionally bridge selected outputs into ContextSeek later

## Why ContextSeek Is The Reference

ContextSeek already has several patterns that map cleanly to `recodex`:

- `DataPlug` separates source-specific ingestion from storage.
- `sync_path()` combines format detection, parsing, hash deduplication, and import
  reporting.
- sync hash and file records prevent repeated imports.
- `FileWatcher` calls the same sync path as manual import.
- `skill_export.py` materializes skills as `out_dir/<slug>/SKILL.md` with a manifest
  so user-authored files are not accidentally deleted.

`recodex` should reuse these ideas, not their full implementation.

## Product Scope

### In Scope

- Preserve current CLI behavior for existing commands.
- Introduce importer interfaces for Codex, Claude Code, Cursor, and future tools.
- Move the existing Codex transcript parsing behind the importer interface.
- Add an incremental sync layer shared by `scan`, `import`, and future `watch`.
- Add export primitives with clear conflict handling.
- Add a copied-and-adapted React dashboard shell with a local JSON API.
- Add a future ContextSeek bridge boundary, without requiring ContextSeek at runtime.

### Out Of Scope

- No FastAPI server dependency in v0.2.0.
- No long-running daemon as the first implementation slice.
- No dependency on ContextSeek as a core runtime dependency.
- No destructive edits to original session files.
- No automatic writes to user `AGENTS.md` or external agent skill directories unless
  the command explicitly asks for it.


## Domain Model

Keep the current `SessionRecord`, `TranscriptEvent`, and `ParsedTranscript`
models as the canonical internal representation.

Add small models for import/sync/export behavior:

```python
@dataclass(frozen=True)
class ImportSource:
    name: str                  # codex, claude_code, cursor
    path: Path
    scope: str | None = None


@dataclass(frozen=True)
class ImportReport:
    source: str
    scanned: int
    imported: int
    skipped: int
    failed: int
    errors: tuple[str, ...]


@dataclass(frozen=True)
class FileSyncRecord:
    source: str
    path: str
    mtime: float
    content_hash: str
    last_seen_at: str
```

`recodex` does not need to adopt ContextSeek `ContextItem` internally. A bridge can
convert recodex outputs into ContextSeek items later.

## Importer Interface

Use a small protocol instead of a heavy plugin framework:

```python
class SessionImporter(Protocol):
    name: str
    default_roots: tuple[Path, ...]
    supported_extensions: tuple[str, ...]

    def discover(self, roots: Iterable[Path]) -> list[Path]:
        ...

    def parse_file(self, path: Path) -> ParsedTranscript:
        ...
```

Expected importers:

- `CodexImporter`: wraps the current `transcripts.py` JSONL/txt parser.
- `ClaudeCodeImporter`: parses Claude Code transcript/log files and normalizes
  source provenance to `claude-code`.
- `CursorImporter`: parses Cursor chat exports or workspace storage.
- Future: `OpenClawImporter`, `HermesImporter`, GitHub Actions logs.

The first v0.2.0 slice only needs `CodexImporter` end to end.

## Sync Layer

Add a shared sync layer inspired by ContextSeek `sync_path()`:

- discover files through the importer
- skip unchanged files by path + mtime
- hash changed files to detect content-equivalent updates
- parse only files that pass the sync gate
- save parsed transcripts through existing `save_transcript`
- record import run counts and errors

Suggested tables:

```sql
CREATE TABLE IF NOT EXISTS sync_files (
  source TEXT NOT NULL,
  path TEXT NOT NULL,
  mtime REAL NOT NULL,
  content_hash TEXT NOT NULL,
  last_seen_at TEXT NOT NULL,
  PRIMARY KEY (source, path)
);

CREATE TABLE IF NOT EXISTS import_runs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  source TEXT NOT NULL,
  root_path TEXT NOT NULL,
  scanned INTEGER NOT NULL,
  imported INTEGER NOT NULL,
  skipped INTEGER NOT NULL,
  failed INTEGER NOT NULL,
  started_at TEXT NOT NULL,
  finished_at TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS import_errors (
  run_id INTEGER NOT NULL,
  path TEXT NOT NULL,
  message TEXT NOT NULL,
  FOREIGN KEY (run_id) REFERENCES import_runs(id) ON DELETE CASCADE
);
```

Existing `sessions.raw_hash` and `raw_session_files` can remain during migration.
The new tables should be additive and backward-compatible.

## CLI Design

Existing commands stay valid:

```bash
recodex scan [paths...]
recodex import [paths...]
recodex retro latest
recodex patterns --since 30d
recodex improvements propose --since 30d
recodex export skills
```

Add explicit source support:

```bash
recodex import --source codex ~/.codex/sessions
recodex import --source claude-code ~/.claude/projects
recodex import --source cursor ~/Library/Application\ Support/Cursor/User/workspaceStorage
recodex import --source auto ~/ai-sessions
```

Add sync reporting:

```bash
recodex import --source codex ~/.codex/sessions --dry-run
recodex import --source codex ~/.codex/sessions --limit 20
```

Watcher commands:

```bash
recodex watch add --source codex --path ~/.codex/sessions --scope default
recodex watch list
recodex watch run
recodex watch status
recodex watch edit 1 --disable
recodex watch delete 1
```

`watch run` starts as a single sync pass over enabled watch sources. A watchdog
dependency should remain optional.

## Export Design

Move export logic out of `reports.py` into focused exporter modules.

### Skill Export

Reference behavior from ContextSeek:

- write `out_dir/<slug>/SKILL.md`
- keep `.recodex-export.json` manifest
- skip unchanged files by content hash
- never prune directories not owned by recodex
- handle conflicts explicitly

CLI:

```bash
recodex export skills --out ~/.codex/skills
recodex export skills --target codex
recodex export skills --target cursor
recodex export skills --target project
recodex export skills --target last
recodex export skills --on-conflict skip
recodex export skills --on-conflict overwrite
recodex export skills --on-conflict rename
```

Default policy:

- only export `accepted` improvements
- if no accepted improvements exist, print a clear message
- do not silently fall back to proposed improvements

### Destination Shortcuts

Destination resolution should be local and transparent:

```text
project   -> ./.agents/skills
codex     -> $CODEX_HOME/skills or ~/.codex/skills
cursor    -> ./.cursor/rules or another documented Cursor-compatible target
custom    -> --out PATH
```

If a target path is uncertain, expose it as a configured default rather than
hard-coding guesses.

## ContextSeek Bridge

The bridge should be optional and isolated.

Possible command:

```bash
recodex bridge contextseek --scope me/ai-coding --since 30d
```

Bridge mapping:

```text
retro report              -> ContextSeek document / knowledge item
accepted improvement      -> ContextSeek skill or knowledge item
repeated failure pattern  -> ContextSeek knowledge item
session timeline event    -> ContextSeek trace_extraction item
```

The bridge must not be required for normal `recodex` usage.

## Implementation Plan

### Phase 1: Importer Boundary

Goal: introduce the abstraction without changing behavior.

Tasks:

- Add `importers/base.py`.
- Add `importers/codex.py`.
- Move or wrap current `transcripts.py` parsing as `CodexImporter`.
- Keep `transcripts.py` as a compatibility facade.
- Update `scan` and `import` commands to use the registry.
- Keep all current tests green.

### Phase 2: Sync Records

Goal: make import idempotent and observable.

- Add file hash helpers.
- Add sync gate before parsing.
- Return `ImportReport` from scan/import.
- Surface dry-run counts in CLI.
- Add tests for unchanged mtime, changed mtime same hash, parse failure, and duplicate run.

### Phase 3: Exporter Split

- Move current `write_skill_exports` behavior behind the new exporter.
- Add manifest and conflict policies.
- Stop falling back from accepted to proposed improvements for skill export.

Tasks:

### Phase 6: Dashboard Workbench

Goal: expose the v0.2.0 import, report, analysis, preview, and export workflows

## Testing Strategy

Keep the current `unittest` style and no-runtime-dependency baseline.

Required test categories:

- importer registry resolution
- Codex importer compatibility with existing fixtures
- sync gate behavior
- import run and error recording
- skill export path resolution
- skill export conflict policies
- accepted-only export behavior
- report generation API and report content API
- artifact preview and artifact export API
- CLI smoke tests for old and new command forms

Property-style tests are useful later for importer robustness, but not required for
the first v0.2.0 slice.

## Migration Rules

- Existing databases must keep working.
- New tables are additive.
- Existing CLI commands must keep their output shape where tests depend on it.
- Existing docs should not be rewritten during the first code slice.
- `transcripts.py` can become a facade, but should not disappear in v0.2.0.
- Do not add mandatory dependencies for watcher support.

## Success Criteria

v0.2.0 is complete when:

- `recodex scan` and `recodex import` use the importer registry.
- Codex import behavior remains compatible with v0.1.x.
- Re-running import on unchanged files skips work using sync records.
- Import runs report scanned/imported/skipped/failed counts.
- Skill export writes one or more valid `SKILL.md` directories.
- Skill export has explicit conflict behavior.
- Skill export only exports accepted improvements by default.
- Claude Code import has parser tests.
- Watch sources can be persisted, disabled, deleted, and synced through the shared
  sync layer.
- The dashboard can generate and preview HTML/Markdown/JSON reports.
- The dashboard can preview and export `SKILL.md` and Markdown artifacts after
  candidate review.
- The full test suite passes.

## Open Questions

- Should `recodex` store source-specific raw metadata in SQLite as JSON, or only keep
  normalized event metadata?
- What should the exact Cursor export destination be for `--target cursor`?
- Should ContextSeek bridge be included in v0.2.0, or planned for v0.3.0?
- Should `watch run` grow an optional `--follow` polling mode in v0.2.x, or wait
  for a daemon/server release?

## Recommended First PR

Start with the smallest safe refactor:

```text
PR 1: Importer boundary

- Add importer protocol and registry.
- Add CodexImporter wrapping current parser.
- Update scan/import to call the registry.
- Keep all behavior and reports unchanged.
- Add tests proving old scan/import still work.
```

This creates the extension point for everything else without committing to watcher,
ContextSeek, dashboard, or new storage behavior too early.
